# UXガイドライン: ウィンドウ・インタラクション

## 1. モーダルウィンドウの振る舞い

特定のタスクを完了させるための、一時的な中断を伴うウィンドウ。

- **視覚誘導**: メインウィンドウにオーバーレイ（シャドウ）をかけ、注意をモーダルに集中させる。
- **出現時**: フェードインと同時に、わずかなスケールアップ（中央からフワッと浮き上がる感覚）を伴う。
- **フォーカス管理**:
  - `Tab` キーによる移動はウィンドウ内の要素間をループさせ、背後へフォーカスを逃さない（フォーカストラップ）。
  - 表示時に最初の入力項目に自動フォーカスする。
- **終了アクション**:
  - **シャドウ領域のクリック**: 保存が必要な画面以外では、ウィンドウ外のクリックで破棄・終了を可能にする。
  - **キーボード操作**: `ESC` キーでキャンセルまたは終了とする。

## 2. モーダレスウィンドウの振る舞い

メインウィンドウの作業と並行して利用する補助的なウィンドウ。

- **視覚的非介入**: オーバーレイは表示しない。背面の情報を読みながら作業できる状態を維持する。
- **出現時**: フェードインのみ。執筆を妨げないよう、控えめな出現とする。
- **フォーカス管理**: フォーカストラップは行わない。クリックによって自在にフォーカスを行き来できるようにする。
- **終了アクション**:
  - タイトルバーのクローズボタン。
  - ウィンドウにフォーカスがある状態での `ESC` キー。
- **連動動作**: メインウィンドウが閉じられた場合、すべてのモーダレスウィンドウも自動的に閉じる。
