# 機能仕様：YAML フロントマター

## 1. 概要

Markdown ファイルの先頭にある YAML 形式のメタデータブロックを認識し、プレビューの制御や属性の自動更新を行う。

## 2. プレビュー制御

フロントマターに含まれるメタデータに基づき、プレビュー画面の表示内容を動的に制御する。

#### YAML ブロックの非表示

Markdown ファイルの冒頭に記述された生の YAML データ（`---` で囲まれたブロック）は、プレビュー画面では一切表示しないものとする。これらは内部的な属性情報として処理し、文書の本文からは除外する。

#### 文書タイトルの自動挿入

システム設定において「プレビューにタイトルを表示」が有効な場合、フロントマターの `title` プロパティの値をプレビューの最上部に独立した要素として自動的に挿入する。この要素は Markdown 本文の H1 要素とは別に管理し、CSS エディタの「タイトル」タブでのカスタマイズ対象とする。なお、`title` が空欄または未定義の場合は、このタイトル要素自体を生成しない。

## 3. 予約プロパティ

アプリケーションが特定の振る舞いを決定するために使用する、予約済みのプロパティ群を以下に定義する。

- **`title`**: 文書のタイトル。プレビュー表示設定が ON の場合、表示対象とする
- **`css`**: 適用する CSS ファイル名。指定がある場合、ドキュメント展開時に自動的に選択・適用する
- **`quote_style`**: 引用ブロックのプリセットスタイル (`simple`, `card`, `topbottom`, `modern`)
- **`quote_icon`**: 引用符アイコンの表示制御 (`true` / `false`)
- **`syntax_highlight`**: コードハイライトのテーマ。システム設定よりも優先して適用する

## 4. 日付・テンプレート管理

文書のライフサイクルを管理するため、特定の日付属性を自動的に更新する。

- **作成日 (`created`)**: 新規保存時において現在日時を自動挿入する
- **更新日 (`updated`)**: 上書き保存時において、既存のプロパティが存在する場合のみ現在日時を自動更新する
- **テンプレート**: 新規作成時において、設定されたフロントマターを含むテンプレートを自動挿入する

## 5. 編集インターフェース

画面右側のサイドパネル下部に、フロントマター編集専用のインターフェースを配置する。ここではキーと値のペアをリスト表示し、主要な属性については未入力時にプレースホルダーを表示することでユーザーの入力を支援する。
