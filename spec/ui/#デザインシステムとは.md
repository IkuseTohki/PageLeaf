# デザインシステムとは

## 1. 目的

デザインシステムは、単なる「見た目の定義」ではなく、ユーザーに対して**一貫した体験（UX）**を提供し、開発者に対して**効率的で迷いのない実装指針**を与えるための「共有言語」である。

本プロジェクトでは、以下の 3 つを主な目的とする。

- **一貫性の保証**: アプリ全体のトーン＆マナーを統一し、プロフェッショナルな品質を維持する。
- **意思決定の高速化**: 新しい UI を作成する際、色、余白、インタラクションの挙動を再定義する必要をなくす。
- **実装品質の担保**: バグの温床になりやすい「色のハードコード」や「場当たり的なスタイル指定」を排除する。

## 2. 三原則

### 2.1. 一貫性 (Consistency)

同じ機能を持つ要素は、アプリのどこにあっても同じ見た目、同じ挙動をしなければならない。

### 2.2. 適応性 (Adaptability)

ライトテーマとダークテーマの両方において、同等の使いやすさと視覚的な快適さを提供しなければならない。

### 2.3. 保守性 (Maintainability)

すべてのデザイン要素は、XAML リソースによって一元管理されなければならない。個別の View (XAML) で「色」や「数値」を決定することを禁止する。

## 3. 運用ルール

- **マスター・オブ・リソース**: 常に XAML リソース (`LightColors.xaml`, `DarkColors.xaml`) を正解とする。
- **スタイルの抽象化**: 個別のコントロールに直接プロパティを書くのではなく、可能な限り `Style` を定義して適用する。
- **変更のプロセス**: 新しいデザインパターンが必要になった場合は、まずこのデザインシステムに定義を追加し、その後に実装を行う。
