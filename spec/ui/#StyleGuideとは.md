# UI Style Guide (UI仕様書) とは

UI Style Guideは、**アプリケーションの視覚的な要素（色、形、動き）を具体的に定義したもの**です。

## 基本方針

- **上位ドキュメントとしての位置付け**: 本ドキュメントは特定のフレームワークやプログラミング言語の実装詳細に依存せず、プロダクト群全体で共有される共通の設計指針（デザイン・システム）として記述します。
- **プラットフォーム非依存**: 「WPFのStyle名」や「特定のライブラリのクラス名」ではなく、「情報の階層」「要素間の距離関係」「視覚的な優先順位」といった抽象的な設計思想をベースに記述します。

## 目的

- 画面上のコンポーネントが、どこで作られても同じ見た目になるようにする。
- 開発者がデザインの細部（px単位の余白など）を迷わず実装できるようにする。
- デザインの変更が必要になった際、影響範囲を一目で確認できるようにする。

## 記述すべきこと

- **具体的な数値**: 余白（Margin/Padding）、サイズ、フォントサイズ。
- **色彩設計**: 使用する色のコードとその用途。
- **アニメーション詳細**: 秒数（ms）、イージング曲線。
- **状態（State）別の定義**: 通常時、マウスホバー時、無効化時などの見た目の変化。
