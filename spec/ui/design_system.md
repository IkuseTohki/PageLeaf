# デザインシステム：意図と規約の調和

本ドキュメントは、アプリケーションの視覚的表現と操作体験（UX）を定義する「デザインの憲法」である。各要素が持つ役割やユーザーに与えるべき心理的影響を地の文で語るとともに、具体的な実装の根拠となるデザイン・トークン（セマンティックな名称と数値）を規定する。

---

## 1. デザイン哲学：静謐な集中と心地よい反応

本プロジェクトが目指すのは、ツールが主役になるのではなく、ユーザーの「コンテンツ（執筆や思考）」を最大限に引き立てるデザインである。

#### 控えめな強調（Soft Emphasis）

情報の階層を示す際、過度な色使いや太い枠線を避け、空間（余白）と微細な色の変化によって状態を伝える。これを「Soft Emphasis」規約と呼び、本デザインシステムの核心とする。

---

## 2. デザイン・トークン（セマンティック命名）

特定の機能やコントロールに依存しない、役割に基づいた命名規則を採用する。これにより、異なるプラットフォーム間でのデザイン共有を容易にする。

### 2.1. カラー・トークン

| トークン名             | 役割                             | PageLeaf での適用例                      |
| :--------------------- | :------------------------------- | :--------------------------------------- |
| **BrandPrimaryBrush**  | 主要なアクション、ブランドの象徴 | 保存ボタン、重要なトグル                 |
| **SurfaceBaseBrush**   | 最背面の下地層                   | ウィンドウ全体の背景                     |
| **SurfaceLayer1Brush** | 一段階浮き上がった面             | サイドパネル、ダイアログの背景           |
| **OutlineBaseBrush**   | 標準的な境界線・区切り線         | テキストボックスの枠線、セパレーター     |
| **TextPrimaryBrush**   | もっとも重要な文字情報           | 本文、メニュー項目                       |
| **TextSecondaryBrush** | 補足、補助的な文字情報           | プレースホルダー、ショートカットキー表示 |

### 2.2. インタラクション・トークン (Soft Emphasis)

| トークン名            | 役割                         | 数値仕様                          |
| :-------------------- | :--------------------------- | :-------------------------------- |
| **StateHoverBrush**   | インタラクションの予兆       | グレー系透過（低不透明度）        |
| **EmphasisLowBrush**  | 控えめな強調（選択状態等）   | アクセント色透過（L:15% / D:35%） |
| **EmphasisHighBrush** | 強い強調（選択時のシンボル） | アクセント色 100%                 |

## 3. スペーシングと形状

一貫したリズムと「手触り」を生み出すための物理的な数値を定義する。

#### スペーシング・グリッド

余白は原則として **4px の倍数**（4, 8, 12, 16, 24, 32）で構成する。

- **標準余白**: 8px または 12px
- **セクション間隔**: 24px 以上

#### 角丸 (CornerRadius)

シャープさと柔らかさのバランスを保つため、以下の値を標準とする。

- **ControlRadius**: **4px**（ボタン、入力フィールド、チェックボックス等）
- **ContainerRadius**: **8px**（パネル、ウィンドウ、ダイアログ等）

---

## 4. 実装における絶対境界（禁止事項）

- **機能名の直接参照禁止**: `IconButtonSelectedBrush` などの機能依存名を排し、必ず `EmphasisLowBrush` などのトークン名を使用すること。
- **色のハードコード禁止**: XAML 内へのカラーコード（HEX 値）の直接記述を禁止する。
- **不透明度のマジックナンバー禁止**: 透明度の計算をコードで行わず、定義されたブラシリソースを使用すること。
