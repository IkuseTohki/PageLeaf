# UI Style Guide: 設定画面 (Settings UI)

## 1. デザインコンセプト

ユーザーがアプリケーションの挙動をカスタマイズするためのインターフェースとして、高い視認性と情報の階層構造を重視する。

## 2. レイアウト構造

一画面に多くの設定項目が並ぶ場合、ナビゲーションとコンテンツエリアを分離する。

- **ナビゲーション**: カテゴリごとの切り替えを左側または上部に配置する。
- **コンテンツエリア**: 選択されたカテゴリの詳細項目を表示する。

## 3. 情報の階層化 (セクション設計)

論理的に関連する項目をグループ化（セクション化）し、情報の区切りを明確にする。

### 3.1. セクションヘッダー

- **強調**: 本文よりも大きなフォント、または境界線を用いて区切りを示す。
- **余白設計**:
  - **最上部**: 画面上端からの余白は最小限とする。
  - **中間セクション**: 前のセクションとの間に十分な余白（目安: 20px 程度）を設け、視覚的な情報の固まりを作る。

## 4. フォーム要素の配置

「何を設定するか（ラベル）」と「現在の値（コントロール）」、および「補足説明（アノテーション）」の関係性を一貫させる。

### 4.1. ラベル

- コントロールの上部または左側に配置し、設定対象を明示する。

### 4.2. 入力コントロール

- 期待される入力値（テキスト、選択肢、数値）に適したコンポーネントを使用する。
- すべての要素で共通の角丸（目安: 4px 程度）を適用し、統一感を持たせる。

### 4.3. 補足情報 (アノテーション)

- 設定内容に専門用語が含まれる場合や、影響範囲を説明する必要がある場合に配置する。
- 補助的な情報であることを示すため、ラベルやコントロールよりも控えめな配色・フォントサイズとする。
- 対象となるコントロールの直近（下部など）に配置し、関連性を維持する。

## 5. 状態のフィードバック

- **変更の検知**: 未保存の変更がある場合、視覚的なインジケータ（ドット、色の変化など）でユーザーに通知する。
- **無効状態**: 現在のモードや他の設定によって操作できない項目は、グレーアウト（低コントラスト化）して明示する。

## 6. アクションボタン

- 「適用」「保存」「キャンセル」などの最終的なアクションは、画面内の一定の場所（例：右下）に固定し、迷わせないようにする。
- 肯定的なアクション（保存など）にはアクセントカラーを適用し、視覚的な優先順位を付ける。
